# S6 Map Overhaul Plan

## Context
Current code has S1 (OPESCO) and S6 (fiords) maps sharing same mechanics:
buffs, territory limits, alliances, CAP marker, same visual style. S6 needs completely separate mechanics and new features.

## Prerequisites / Existing State
- `REGION_DATA` computed from `season` state (`S1_REGION_DATA` vs `S6_REGION_DATA`)
- `TerritoryMapData` interface at `src/AllianceMapManager.tsx:11`
- Season saved/loaded via Firestore field `season` at `users/{userId}/territory_map/main`
- Render logic: paths with `fillOpacity: 0.7` (alliance) or `0.5` (unclaimed), buffs via `PERMANENT_BUFFS`, CAP marker at line 1421
- Alliance sidebar panel at line ~1550 with 8-territory limit
- Edit centers mode exists (`isEditingCenters`, `manualCenterOverrides`)
- Admin check: `userId === import.meta.env.VITE_ADMIN_DISCORD_ID` (from FrankensteinEventTab pattern)

### 4. Devtools Modal (admin only, S6 only)

**New file: `src/DevtoolsModal.tsx`**

Pattern: follow `SaveTemplateModal.tsx` (dark overlay, backdrop blur, border #2a2a3a, inline styles + Tailwind).

Props:
```ts
interface DevtoolsModalProps {
  isOpen: boolean;
  onClose: () => void;
  regionData: RegionData[];       // current season's regions
  territoryLevels: Record<string, number>;
  onLevelChange: (regionId: string, level: number) => void;
  levelPositions: Record<string, RegionCenter>;
  onLevelPositionChange: (regionId: string, pos: {x: number, y: number}) => void;
  buildings: Building[];
  onBuildingAdd: (building: Building) => void;
  onBuildingRemove: (id: string) => void;
  regionCenters: Record<string, RegionCenter>; // for building placement reference
}
```

**Sections:**
- **Territory Levels**: scrollable list of all regions with level input (number, 1-10) and X/Y position inputs
- **Buildings**: list of placed buildings with type, coordinates, delete button. "Add Building" opens mode to click on map.

**Building placement flow:**
- Click "Add Building" button → modal minimizes/closes, cursor changes to crosshair
- Admin clicks a point on the SVG map
- `handleMapClickInEditMode` extended: if `buildingPlacementMode` is active, capture click coordinates and call `onBuildingAdd`
- New building created with `id: crypto.randomUUID()`, `x`, `y`, `type: 'default'`
